# Web-DLP: Повна документація проєкту

**Web-DLP** — це сучасний локальний веб-інтерфейс для потужної утиліти `yt-dlp` та інструментів `FFmpeg`. Проєкт дозволяє зручно завантажувати відео з YouTube у високій якості (аж до MAX/4K), автоматично вбудовувати субтитри та керувати медіафайлами (наприклад, об'єднувати відео з окремими аудіодоріжками) через єдиний мінімалістичний дашборд.

---

## 🚀 Як запустити проект (Автоматичний запуск)

Для максимальної зручності проект містить готові скрипти запуску, які автоматично ініціалізують середовище та відкривають сервер.

### macOS / Linux
1. Переконайтеся, що у вас встановлені **Python 3.8+** та **FFmpeg** (наприклад, через `brew install ffmpeg`).
2. Двічі клацніть файл `start-server-mac.command` (або виконайте `./start-server-mac.command` у терміналі).
3. Скрипт автоматично створить `venv` (якщо його немає), встановить залежності, запустить FastAPI сервер на порті `8080` та виведе посилання.

### Windows
1. Двічі клацніть файл `start-server-windows.bat`.
2. Сервер автоматично налаштує середовище та запуститься.

**Доступ до інтерфейсу:** Перейдіть у браузері за адресою [http://localhost:8080](http://localhost:8080).

## 🛠 Ручне розгортання (Manual Setup)

Якщо ви бажаєте розгорнути проєкт вручну:

1. **Встановіть системні залежності:** `python3`, `ffmpeg`.
2. **Створіть віртуальне середовище:**
   ```bash
   python3 -m venv venv
   source venv/bin/activate
   ```
3. **Встановіть Python-бібліотеки:**
   ```bash
   pip install -r requirements.txt
   ```
4. **Запустіть сервер:**
   ```bash
   cd app
   python -m uvicorn main:app --host 0.0.0.0 --port 8080 --reload
   ```

## ⚙️ Архітектура та Алгоритм роботи

Проєкт побудований за клієнт-серверною архітектурою з підтримкою стрімінгу даних у реальному часі:

- **Frontend (UI)**: Односторінковий Vanilla JS / HTML5 додаток з адаптивним "Dark Mode" інтерфейсом. Взаємодіє з бекендом через Fetch API для команд та **WebSockets** для отримання логів завантаження у стилізовану віртуальну консоль.
- **Backend (FastAPI)**: Легкий та швидкий асинхронний сервер. Керує фоновими задачами (`asyncio`).
- **Модуль завантаження (yt-dlp)**: Використовує Python-модуль `yt_dlp` напряму. **Важлива особливість:** для стабільної роботи без блокувань (HTTP 429) бекенд налаштований на завантаження лише *оригінальної* доріжки субтитрів. Усі завантажені файли автоматично зберігаються у системну папку `~/Downloads`.
- **Модуль FFmpeg**: Надає API (`/api/ffmpeg/replace`) для миттєвої (без перекодування, `copy` кодек) заміни аудіодоріжки у завантаженому відео на будь-який інший аудіофайл із папки `Downloads`.

### Особливості боротьби з блокуваннями YouTube (Update 2026)
У розробці застосовано механізми стабілізації завантажень:
1. **Оптимізовані субтитри**: Для уникнення жорстких блокувань (HTTP 429 Too Many Requests), інтерфейс має єдину галочку "Subtitles". Вона дає команду `yt-dlp` завантажувати **тільки** автогенеровані субтитри мовою оригіналу (без спроб машинного перекладу "на льоту"). Це дозволяє завантажувати субтитри без 429 помилки.
*Примітка щодо JS:* Під час завантаження `yt-dlp` може показувати попередження про відсутність JS Runtime. Поточна базова версія не вимагає його для успішного завантаження більшості відео.
